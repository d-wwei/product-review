# Persona 档案完整模板（数据文件，按需 Read）

> 本文件是 persona 档案的**完整字段定义**。
> `step-5.5-6-personas.md` 中的精简模板(8行)用于主 agent 的调度上下文。
> 当 subagent 需要生成完整 persona 档案、或模式 A 需要沉浸式角色扮演时，
> Read 本文件获取完整字段结构。
>
> **加载时机**：Step 6.0 生成 persona 档案时、模式 A subagent 分派时。
> **不要在 Step 5.5 分析阶段加载**（分析只需精简模板）。

---

## 完整 Persona 档案结构

```markdown
## Persona #{N}: {姓名}（{一句话标签}）

### 基础属性
| 属性 | 值 |
|------|-----|
| 姓名 | {真实感的中文/英文名} |
| 年龄 | {age} |
| 性别 | {gender} |
| 职业 | {profession}（{具体描述，如"三甲医院骨科住院医"而非"医生"}) |
| 教育背景 | {education} |
| 所在地 | {city, country} |
| 收入水平 | {月收入范围 或 年薪范围} |
| 家庭状态 | {单身/已婚/有孩子等} |

### 技术与领域认知
| 维度 | 等级 | 具体表现 |
|------|------|---------|
| 智能手机熟练度 | {1-5} | {如"只会用微信和支付宝"} |
| 产品领域经验 | {1-5} | {如"炒股 3 年，用过同花顺和东方财富"} |
| 同类产品经历 | — | {2-3 个同类 App 及使用深度} |
| 新功能接受度 | {保守/跟随/尝鲜} | {具体描述} |

### 动机与目标
- **核心动机**: {具体场景，不要泛泛}
- **期望解决的问题**: 1. {问题1} 2. {问题2}
- **学习成本预算**: {低/中/高} — {具体描述}
- **付费意愿**: {免费党/按需付费/订阅型}

### 行为模式
| 维度 | 取值 | 说明 |
|------|------|------|
| 注意力模式 | {快速扫描/仔细阅读/目标导向} | {具体描述} |
| 耐心程度 | {低/中/高} | {具体描述} |
| 错误容忍度 | {低/中/高} | {具体描述} |
| 探索风格 | {引导依赖/自主探索/搜索优先} | {具体描述} |
| 使用场景 | {具体场景} | {如"早上通勤地铁上"} |
| 单次使用时长 | {时长范围} | {如"每次不超过 5 分钟"} |

### 性格特征
| 特征 | 取值 | 说明 |
|------|------|------|
| 表达风格 | {直接犀利/温和委婉/技术分析/情绪感受} | {具体描述} |
| 关注重点 | {功能/视觉/效率/性价比} | {优先关注什么} |
| 评价倾向 | {严格挑剔/公正中立/包容宽松} | {描述} |
| 社交倾向 | {独立使用/喜欢讨论/会写评测} | {描述} |

### 个人背景故事（200 字）

{一段叙事性的背景描述，包含：
- 这个人的生活状态
- 为什么会接触这个产品
- 之前用同类产品的经历（好的和坏的）
- 对这类产品的期望和偏见
让 persona 从属性表格变成一个"活生生的人"。}

### 评价倾向参数（内部使用，不展示给用户）
- sentiment_bias: {-1.0 到 1.0}  — 整体评价乐观/悲观倾向
- detail_orientation: {0 到 1.0}  — 关注细节 vs 大局
- comparison_tendency: {0 到 1.0} — 倾向于与竞品比较
- patience_score: {0 到 1.0}     — 耐心分数，影响探索深度
- tech_confidence: {0 到 1.0}    — 技术自信度

### 领域专业知识（如适用）
> 仅领域专家 persona 需要。从 appendix-domain-experts.md 按需注入。
- **领域专家原型**: {对应原型名，或"通用用户"}
- **领域知识注入**: Read appendix-domain-experts.md 对应段落
- **领域专项测试清单**: 同上
```

## 行为指令集模板

> 下方模板在模式 A（自主操控）时注入 subagent prompt，
> 让 persona 在具体场景下有确定性的行为反应。
> 模式 C（评审）时不需要此模板。

```markdown
## {姓名} 的行为指令集

### 首次启动行为
- 看到引导页: {认真看完 / 快速翻过 / 直接跳过}
- 看到权限请求: {全部允许 / 逐个考虑 / 全部拒绝}
- 看到登录页: {直接注册 / 第三方登录 / 先看能不能不登录用}
- 看到首页: {按引导操作 / 先四处看看 / 直奔目标功能}

### 导航行为
- 发现底部 Tab: {从左到右逐个点 / 直找目标 / 先待首页}
- 看到搜索框: {忽略 / 试着搜 / 第一个使用的功能}
- 遇到深层嵌套: {每层都进 / 到第二层返回 / 按标题决定}
- 遇到未知图标: {猜测后点 / 忽略 / 长按看提示}

### 内容消费行为
- 长列表: {看前3条 / 滑到第10条 / 一直到底}
- 详情页: {只看标题摘要 / 完整阅读 / 直看关键数据}
- 图表/数据: {看不懂跳过 / 仔细研究 / 只看涨跌颜色}
- 弹窗/广告: {立即关闭 / 阅读内容 / 有优惠看一下}

### 交互与输入行为
- 表单填写: {尽量少填 / 认真填 / 必填太多就放弃}
- 遇到报错: {再试一次 / 截图退出 / 尝试不同输入}
- 功能卡顿: {等待 / 退出重进 / 直接卸载}
- 需要付费: {立即关闭 / 看价格 / 找免费替代}

### 情绪触发器
| 触发条件 | 情绪 | 行为后果 |
|---------|------|---------|
| {正面: 如"操作一步完成"} | 满意/惊喜 | {继续探索/分享朋友} |
| {负面: 如"找不到返回按钮"} | 困惑/烦躁 | {反复尝试/放弃功能} |
| {强烈负面: 如"闪退丢输入"} | 愤怒 | {卸载/差评} |
| {惊喜: 如"智能推荐很准"} | 超出预期 | {深度使用/推荐朋友} |
```

## 使用指南

### 精简模板 vs 完整模板的使用场景

| 场景 | 使用哪个 | 原因 |
|------|---------|------|
| Step 5.5 分析阶段 | 精简模板(8行) | 只需维度和采样规则 |
| 模式 B/C 评审 | 精简模板 + 动机描述 | 评审不需要行为指令 |
| 模式 A 自主操控 | Read 本文件，生成完整档案 | 角色扮演需要沉浸感 |
| persona-profiles.md 输出 | 完整模板 | 最终产出物供报告引用 |
| subagent prompt 注入 | 精简档案 + 行为指令集 | 控制 prompt 大小 |

### 完整档案的生成流程

```
1. 主 agent 在 Step 5.5 用精简模板确定 persona 属性矩阵
2. Step 6.0 时 Read persona-templates.md，为每个 persona 填充完整档案
3. 完整档案写入 {BASE_DIR}/persona-profiles.md（最终产出物）
4. 分派 subagent 时：
   - 模式 A: 注入完整档案 + 行为指令集（从 persona-profiles.md Read）
   - 模式 C: 只注入精简档案 + 动机描述（从精简模板内联）
```
